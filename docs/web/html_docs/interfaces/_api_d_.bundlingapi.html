bundleFeature

bundleFeature: function
  • Defined in api.d.ts:167

Bundles A WebIDE Feature, This includes JS Sources, i18n and JSON metadata. It does not include bundling of any UI5 resources. Use tools provided by UI5 to bundle those...

returns a promise with an object. The object properties: outDir: output directory containing the bundled files. The output directory's name may not be known in advance as it may include a timestamp when the caching option is enabled. By exposing the exact name, custom post processing steps may be more easily implemented by the user.

      • target: string

        • Optional cleanOutDir?: boolean

          Should the output directory be cleaned prior to the bundling. This is enabled by default, if you have custom code which creates additional packaged artifacts to this directory, disabling this option should be considered. default value is true

        • Optional enableCaching?: boolean

          bundle into 'outDir/[VERSION]_[TIMESTAMP]/' instead of 'outDir/' and create a wrapper package.json in outDir/package.json to enable caching support. default value is true.

        • Optional javaScriptOpts?: object

          SubOptions for the javaScript bundling phase.

          • Optional ignore?: string | string[]

            Ignored glob patterns, by default all JS resources (recursively) in the directory of the plugin.json will be bundled. However, some special edge cases may require exclusion.

            https://github.com/isaacs/node-glob for details on valid patterns syntax.

            Will only log validations errors on the bundled artifacts Instead of throwing an error (rejecting the promise). Should only be enabled if you know what you are doing... @link {https://github.com/SAP/webide-client-tools/blob/master/FAQ.md#VALIDATE

          • Optional optimizeOptions?: Object

            custom configurations for the require.js optimizer.

            see

        • Optional metadataOpts?: object

          SubOptions for the JSON metadata bundling phase.

          • Optional pluginsPrefix?: string

            Prefix to apply to all plugins paths, for example "quickstart" --> "w5g/quickstart"

          Output directory for the bundle. default value is 'dist'.
